# 动物棋子 2.5D 立体化修改计划

## 目标
把当前平面的动物棋子改成“动物站在一个小平台上”的立体视觉效果，同时保持现有棋盘规则、点击、移动、吃子和胜负逻辑不变。

## 改造思路
优先做 2.5D 假 3D，而不是直接切到完整 3D。

原因：
- 现有项目已经是 Cocos Creator 的 2D UI 结构
- 现有棋盘与棋子逻辑已经完整
- 假 3D 能以较小成本获得明显的立体感
- 真 3D 需要重做更多资源和场景配置，投入更大

## 建议效果结构
每个棋子建议拆成以下视觉层：
- `Shadow`：底部阴影
- `Base` 或 `Platform`：小平台
- `Animal`：动物本体
- `Label`：名称或等级文字

如果当前 prefab 已经有 `Base` 和 `Animal`，就只补一个 `Shadow`，不必重建整个结构。

## 实施步骤

### 1. 先只改棋子表现，不动引擎
保持以下内容不变：
- `LocalEngine`
- 走子规则
- 吃子规则
- 胜负判断

先集中修改：
- `PieceView.ts`
- `Piece.prefab`
- 棋子资源图

当前进度：
- `PieceView.ts` 已经先完成了一版重构，开始支持“平台 + 阴影 + 动物”的分层表现
- 现有的棋子规则和 `BoardView` 对外接口保持不变

### 2. 重构棋子预制体层级
在 `Piece.prefab` 中把棋子拆成多个视觉节点，推荐层级如下：

```text
Piece
  Shadow
  Base / Platform
  Animal
  Label
```

如果后续要更精细，可以继续加：
- `Rim`：底座边缘高光
- `Glow`：选中时外发光

当前进度：
- 代码层已经预留了 `Shadow` 层的逻辑
- prefab 结构还没正式补齐，后续可以按这个层级继续整理

### 3. 准备平台资源
补一版更像“棋子底座”的图片资源。

建议底座图满足这些特点：
- 顶面稍亮
- 侧面有厚度
- 边缘有轻微高光
- 整体偏圆润，不要太扁平

如果暂时没有新图，可以先复用现有底图，但最终建议单独出一张平台图。

### 4. 改 `PieceView.ts` 的初始化
初始化时分别设置底座、阴影和动物的位置与缩放。

推荐做法：
- 动物稍微抬高一点
- 底座略大于动物本体
- 阴影放在最底部，半透明，略微压扁
- 红方和蓝方只做轻微配色，不要把动物本体染得过重

当前进度：
- `PieceView.ts` 已经完成改造
- 现在棋子支持独立的阴影、底座、动物与标签布局
- 选中、移动、吃子动画也已经切换成更立体的表现

### 5. 改动画表现
选中、移动、吃子三个场景都做一点立体动效。

选中时：
- 动物轻微弹起
- 底座略微变宽或压缩
- 阴影略微变淡并放大一点

移动时：
- 让棋子有轻微起落感
- 不要只做直线平移

吃子时：
- 攻击方先冲一下
- 被吃方缩小并后退消失
- 保持平台和阴影一起退出

当前进度：
- 选中动效已改成轻微弹起和阴影收缩
- 移动动效已改成带轻微起落感的过渡
- 吃子动效已改成整体缩小并伴随阴影消失

### 6. 做缩放适配
棋盘容器现有的缩放逻辑保留。

只要保证：
- 平台不会因为缩放显得太薄
- 阴影不会糊成一团
- 动物和底座的层次在不同分辨率下都清楚

### 7. 最后做视觉微调
重点调这些参数：
- 底座宽高比
- 动物与底座的上下间距
- 阴影大小和透明度
- 选中时弹跳幅度
- 移动时的落地速度

## 推荐执行顺序
1. 先改 prefab 层级
2. 再改 `PieceView.ts`
3. 再补平台资源
4. 最后调动画和尺寸

## 风险点
- 平台图太扁，会显得还是平面
- 阴影过重，会显得脏
- 动物和底座间距不对，会像漂浮或嵌进去
- 一次改太多，会不好定位是哪一层出了问题

## 预期结果
- 棋子看起来像“动物站在小平台上”
- 现有规则完全不变
- 点击、移动、吃子逻辑不受影响
- 改动范围集中在棋子显示层
