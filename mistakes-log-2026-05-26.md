# 失误记录

记录时间：2026-05-26

这份文档只记录这次在“动物走路动画”处理过程中犯过的错误，方便后续回看和避免重复踩坑。

## 1. 裁剪错误

- 一开始直接把动物走路图做成帧图时，没有先统一画布和中心点，导致有些帧把动物身体裁掉了。
- 不同动物的左右留白不一致，有的偏左，有的偏右，放到棋盘上之后视觉位置不统一。
- 没有先检查完整序列里每一帧的边界，导致部分帧的头、尾、腿被切掉。

## 2. 资源导入错误

- 把图片放进 `textures/animals_walk` 后，没有先确认 Cocos 实际可加载的资源路径和资源类型。
- 早期尝试过按 `resources.loadDir` 和文件名匹配来加载，但资源命名和导入后的对象名不完全一致，导致运行时拿不到帧。
- 没有先确认 `SpriteFrame` 是否已经正确导入，就直接在代码里做动画循环，结果出现 `0` 帧可用的情况。

## 3. 命名和对应关系错误

- 走路帧与棋子动物类型的对应关系没有一开始就固定下来，导致类型和资源名字可能对不上。
- 没有先统一好“原始图文件名 -> 导入后资源名 -> 动物类型编号”的映射，排查时浪费了时间。

## 4. 代码实现上的错误

- 过早改动了资源加载方式，导致 runtime 里反复切换 `SpriteFrame` / 纹理加载方案，增加了不稳定因素。
- 没有先把资源加载日志和文件名一条条对照，导致问题定位慢。
- 动画逻辑本身其实没大问题，真正的问题主要出在素材裁剪和资源导入，而不是棋子选中逻辑。

## 5. 以后要先做的检查

- 先保证每一帧的画布尺寸一致。
- 先保证动物在画面中心对齐，再导入到引擎。
- 先确认资源导入后的名称、类型、路径，再写加载逻辑。
- 先用日志确认每种动物实际加载到了多少帧，再判断动画逻辑是否正确。
